- React.js is a JavaScript library used to build interactive and dynamic user interfaces (UI), mainly for web applications. 
- It helps developers create fast, efficient, and reusable UI components.

<h2 style= "color:blue">Why is React.js Used?</h2>

Imagine building a big house with small, reusable building bricks instead of crafting everything from scratch. That’s exactly what React does for web applications!

✅ Fast & Efficient – Only updates the necessary parts of the page (using the Virtual DOM) instead of reloading the whole page.

✅ Reusable Components – You can create one component (like a button) and use it multiple times across the app.

✅ Easy to Maintain – Code is structured, making large projects easier to manage.

✅ Used by Big Companies – Facebook, Instagram, Netflix, Airbnb, and many others use React.

<h2 style= "color:blue">Long-Term Example to Remember</h2>

🔹 Think of React like a restaurant kitchen. Instead of cooking every dish from scratch, chefs use pre-prepared ingredients (components) to make the food faster.

🔹 Similarly, React developers use components (like buttons, forms, and navigation bars) to build web apps quickly without rewriting code.

<h2 style= "color:blue">How is React Useful?</h2>

📌 For Developers → Makes coding easier with reusable components and better performance.

📌 For Users → React apps load faster, feel smooth, and provide a great experience.

📌 For Companies → Saves time and cost by improving development speed and maintainability.

 <h2 style= "color:blue">Components</h2>

- A component in React is like a small, reusable piece of code that can be used to build elemts on page. 
- Instead of writing the entire webpage in one file, we divide it into smaller parts (components).

## 1️⃣ Why Use Components?

✅ Reusability – Write once, use anywhere.

✅ Organized Code – Makes big projects easier to manage.

✅ Easier Maintenance – Fix one component instead of editing multiple files.


## 2️⃣ Types of React Components
React has two types of components:

1️⃣ Functional Components (✅ Modern, recommended)

2️⃣ Class Components (❌ Old, not used in new projects)

💡 We will use Functional Components because they are simpler and better!

- A Functional Component is just a JavaScript function that returns JSX (HTML-like code).


<h2 style= "color:blue">State in Component</h2>

- Until now, our React components have been static—they just display fixed text. But what if we want them to change dynamically?

- That’s where State comes in!
  
👉 State allows React components to store and manage data that can change over time.

## 1️⃣ Why Do We Need State?

- Without state, a webpage is like a printed newspaper—the content doesn’t change.
- With state, it’s like a news website—the content updates automatically!

**Example:** A Counter App
- Imagine a button that increases a number every time we click it.
- Without state, the number won’t update.
- With state, React remembers and updates the number.

## 2️⃣ How to Use State?
React provides a special Hook called useState to manage state.

📌 Steps to Use State in a Component

1️⃣ Import useState from React

2️⃣ Create a state variable using useState

3️⃣ Update the state when needed
